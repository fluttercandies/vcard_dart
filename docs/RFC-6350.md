# RFC 6350: vCard Format Specification (vCard 4.0)

> **Document Type**: Standard Summary  
> **RFC Number**: 6350  
> **Category**: Standards Track  
> **Status**: Current Standard  
> **Date**: August 2011  
> **Author**: S. Perreault (Viagenie)  
> **Obsoletes**: RFC 2425, RFC 2426, RFC 4770  
> **Updates**: RFC 2739

## Overview

RFC 6350 defines the **vCard 4.0** format specification. It merges and supersedes RFC 2425 (MIME Directory), RFC 2426 (vCard 3.0), and RFC 4770 (IMPP extension).

## Key Changes from vCard 3.0

### Structural Changes
- vCard is now a stand-alone format (not just MIME type)
- UTF-8 is the **only** allowed character set
- IANA registration process for new elements
- Proper MIME type registration (`text/vcard`)

### Removed Features
- `CONTEXT` and `CHARSET` parameters removed
- `NAME`, `MAILER`, `LABEL`, `CLASS` properties removed
- "intl", "dom", "postal", "parcel" TYPE values for ADR removed
- In-line vCards (AGENT property) no longer supported

### New Properties
- `KIND` - Entity type (individual, group, org, location)
- `GENDER` - Sex and gender identity
- `LANG` - Communication languages
- `ANNIVERSARY` - Marriage/relationship date
- `XML` - Extended XML-encoded data
- `CLIENTPIDMAP` - PID source mapping
- `MEMBER` - Group member references
- `RELATED` - Relationships to other entities

### New Parameters
- `PREF` - Preference level (1-100)
- `ALTID` - Alternative representations
- `PID` - Property instance identifier
- `MEDIATYPE` - Media type hint for URIs
- `CALSCALE` - Calendar system
- `SORT-AS` - Sorting string

## Key Requirements

### Required Properties
- `VERSION` - **MUST** be "4.0", **MUST** appear immediately after `BEGIN:VCARD`
- `FN` - Formatted Name (cardinality: 1*)

### Structure
```
BEGIN:VCARD
VERSION:4.0
... properties ...
END:VCARD
```

## Property Cardinality

| Symbol | Meaning |
|--------|---------|
| `1` | Exactly one instance per vCard MUST be present |
| `*1` | Exactly one instance per vCard MAY be present |
| `1*` | One or more instances per vCard MUST be present |
| `*` | One or more instances per vCard MAY be present |

## Value Data Types

| Type | Description | Format |
|------|-------------|--------|
| `TEXT` | Human-readable text | Escaped string |
| `URI` | Uniform Resource Identifier | RFC 3986 URI |
| `DATE` | Calendar date | `YYYYMMDD`, `YYYY-MM`, `--MMDD`, `---DD` |
| `TIME` | Time of day | `HHMMSS`, `HHMM`, `HH`, with optional zone |
| `DATE-TIME` | Combined date and time | `YYYYMMDDTHHMMSS` |
| `DATE-AND-OR-TIME` | DATE-TIME, DATE, or TIME | `T` prefix for stand-alone time |
| `TIMESTAMP` | Complete date-time | `YYYYMMDDTHHMMSS[Z\|+/-HHMM]` |
| `BOOLEAN` | Boolean value | `TRUE` or `FALSE` (case-insensitive) |
| `INTEGER` | Signed integer | 64-bit range |
| `FLOAT` | Real number | No scientific notation |
| `UTC-OFFSET` | UTC offset | `[+/-]HHMM` |
| `LANGUAGE-TAG` | Language identifier | RFC 5646 tag |

## Properties Reference

### General Properties

| Property | Cardinality | Value Type | Description |
|----------|-------------|------------|-------------|
| `BEGIN` | 1 | text | Must be "VCARD" |
| `END` | 1 | text | Must be "VCARD" |
| `SOURCE` | * | uri | Directory source |
| `KIND` | *1 | text | Entity type: `individual`, `group`, `org`, `location` |
| `XML` | * | text | Extended XML data |

### Identification Properties

| Property | Cardinality | Value Type | Description |
|----------|-------------|------------|-------------|
| `FN` | 1* | text | Formatted name (REQUIRED) |
| `N` | *1 | structured | Family;Given;Additional;Prefix;Suffix |
| `NICKNAME` | * | text-list | Nickname(s) |
| `PHOTO` | * | uri | Photo URI (data: URI for inline) |
| `BDAY` | *1 | date-and-or-time/text | Birthday |
| `ANNIVERSARY` | *1 | date-and-or-time/text | Anniversary |
| `GENDER` | *1 | structured | Sex;Gender-identity |

**GENDER Sex Values**: `M` (male), `F` (female), `O` (other), `N` (none/not applicable), `U` (unknown)

### Delivery Addressing Properties

| Property | Cardinality | Value Type | Description |
|----------|-------------|------------|-------------|
| `ADR` | * | structured | PO Box;Ext;Street;City;Region;Postal;Country |

**ADR Parameters**: `LABEL`, `GEO`, `TZ` can be applied  
**Note**: Post office box and extended address SHOULD be empty for interoperability

### Communications Properties

| Property | Cardinality | Value Type | Description |
|----------|-------------|------------|-------------|
| `TEL` | * | uri/text | Telephone (prefer `tel:` URI) |
| `EMAIL` | * | text | Email address |
| `IMPP` | * | uri | Instant messaging URI |
| `LANG` | * | language-tag | Communication language |

**TEL TYPE Values**: `text`, `voice` (default), `fax`, `cell`, `video`, `pager`, `textphone`

### Geographical Properties

| Property | Cardinality | Value Type | Description |
|----------|-------------|------------|-------------|
| `TZ` | * | text/uri/utc-offset | Timezone (prefer Olson DB names) |
| `GEO` | * | uri | Geographic position (prefer `geo:` URI) |

### Organizational Properties

| Property | Cardinality | Value Type | Description |
|----------|-------------|------------|-------------|
| `TITLE` | * | text | Job title |
| `ROLE` | * | text | Role/function |
| `LOGO` | * | uri | Organization logo |
| `ORG` | * | structured | Org;Unit1;Unit2;... |
| `MEMBER` | * | uri | Group member (only for KIND=group) |
| `RELATED` | * | uri/text | Related entity |

**RELATED TYPE Values**: `contact`, `acquaintance`, `friend`, `met`, `co-worker`, `colleague`, `co-resident`, `neighbor`, `child`, `parent`, `sibling`, `spouse`, `kin`, `muse`, `crush`, `date`, `sweetheart`, `me`, `agent`, `emergency`

### Explanatory Properties

| Property | Cardinality | Value Type | Description |
|----------|-------------|------------|-------------|
| `CATEGORIES` | * | text-list | Tags/categories |
| `NOTE` | * | text | Comments |
| `PRODID` | *1 | text | Product identifier |
| `REV` | *1 | timestamp | Last revision |
| `SOUND` | * | uri | Pronunciation audio |
| `UID` | *1 | uri/text | Unique identifier (prefer UUID URN) |
| `CLIENTPIDMAP` | * | structured | PID;URI mapping |
| `URL` | * | uri | Website |
| `VERSION` | 1 | text | Must be "4.0" (REQUIRED) |

### Security Properties

| Property | Cardinality | Value Type | Description |
|----------|-------------|------------|-------------|
| `KEY` | * | uri/text | Public key/certificate |

### Calendar Properties (from RFC 2739)

| Property | Cardinality | Value Type | Description |
|----------|-------------|------------|-------------|
| `FBURL` | * | uri | Free/busy URL |
| `CALADRURI` | * | uri | Calendar scheduling address |
| `CALURI` | * | uri | Calendar URL |

## Parameters

### LANGUAGE (Section 5.1)
```
ROLE;LANGUAGE=tr:hoca
```

### VALUE (Section 5.2)
```
VALUE=text|uri|date|time|date-time|date-and-or-time|timestamp|boolean|integer|float|utc-offset|language-tag
```

### PREF (Section 5.3)
Integer 1-100, lower = more preferred
```
TEL;PREF=1:tel:+1-555-555-5555
EMAIL;PREF=2:secondary@example.com
```

### ALTID (Section 5.4)
Groups alternative representations of the same logical property
```
N;ALTID=1;LANGUAGE=jp:山田;太郎;;;
N;ALTID=1;LANGUAGE=en:Yamada;Taro;;;
```

### PID (Section 5.5)
Property instance identifier for synchronization
```
TEL;PID=1.1:tel:+1-555-555-5555
```

### TYPE (Section 5.6)
Common values: `work`, `home`
```
TEL;TYPE="work,voice":tel:+1-555-555-5555
```

### MEDIATYPE (Section 5.7)
Hint for URI content type
```
KEY;MEDIATYPE=application/pgp-keys:ftp://example.com/keys/jdoe
```

### CALSCALE (Section 5.8)
Calendar system (default: `gregorian`)
```
BDAY;CALSCALE=gregorian:19850412
```

### SORT-AS (Section 5.9)
```
N;SORT-AS="Harten,Rene":van der Harten;Rene;J.;Sir;R.D.O.N.
ORG;SORT-AS="Example":The Example Organization
```

### GEO (Section 5.10)
Geographic position for ADR
```
ADR;GEO="geo:12.3457,78.910":;;123 Main St;City;ST;12345;US
```

### TZ (Section 5.11)
Timezone for ADR
```
ADR;TZ="America/New_York":;;123 Main St;City;ST;12345;US
```

## Escaping Rules

### Characters Requiring Escape

| Character | Escape Sequence | Context |
|-----------|-----------------|---------|
| `\` (backslash) | `\\` | Always |
| `,` (comma) | `\,` | Always (even if property doesn't allow multiple values) |
| `;` (semicolon) | `\;` | In compound property fields |
| Newline | `\n` or `\N` | Always |

### Escaping Contexts
1. **COMMA**: Always escape in values (for consistency)
2. **SEMICOLON**: Escape in compound property fields
3. **BACKSLASH**: Always escape
4. **NEWLINE**: Use `\n` or `\N` sequence

## Line Folding

- Lines SHOULD be folded to max 75 octets
- Fold by inserting CRLF + single whitespace (space or tab)
- Multi-octet characters MUST remain contiguous
- Unfolding removes CRLF + following whitespace completely

```
NOTE:This is a very long description that needs to be
  folded onto multiple lines.
```

## Date/Time Formats

### DATE
```
19850412        ; Complete: April 12, 1985
1985-04         ; Year and month
1985            ; Year only
--0412          ; Month and day (year unknown)
---12           ; Day only (month and year unknown)
```

### TIME
```
102200          ; 10:22:00
1022            ; 10:22
10              ; 10 o'clock
-2200           ; 22 minutes past the hour
--00            ; 0 seconds past the minute
102200Z         ; UTC
102200-0800     ; With offset
```

### DATE-TIME
```
19961022T140000
--1022T1400
---22T14
```

### DATE-AND-OR-TIME
```
19961022T140000    ; Full date-time
19850412           ; Date only
T102200            ; Time only (T prefix required)
T10                ; Hour only
```

### TIMESTAMP
```
19961022T140000    ; Local time
19961022T140000Z   ; UTC
19961022T140000-05 ; With UTC offset
19961022T140000-0500
```

## ABNF Grammar (Key Excerpts)

```abnf
vcard = "BEGIN:VCARD" CRLF
        "VERSION:4.0" CRLF
        1*contentline
        "END:VCARD" CRLF

contentline = [group "."] name *(";" param) ":" value CRLF

N-value = list-component 4(";" list-component)
          ; Family; Given; Additional; Prefix; Suffix

ADR-value = ADR-component-pobox ";" ADR-component-ext ";"
            ADR-component-street ";" ADR-component-locality ";"
            ADR-component-region ";" ADR-component-code ";"
            ADR-component-country

GENDER-value = sex [";" text]
sex = "" / "M" / "F" / "O" / "N" / "U"
```

## Synchronization (Section 7)

### UID Property
- Used to match multiple instances of the same vCard
- Recommended: `urn:uuid:` URN namespace

### PID Parameter
- Identifies specific property instances
- Format: `local-id[.source-id]`
- Used with CLIENTPIDMAP for global identification

### CLIENTPIDMAP Property
```
TEL;PID=3.1:tel:+1-555-555-5555
CLIENTPIDMAP:1;urn:uuid:53e374d9-337e-4727-8803-a1e9c14e0556
```

## Examples

### Minimal vCard 4.0
```
BEGIN:VCARD
VERSION:4.0
FN:John Doe
END:VCARD
```

### Complete vCard 4.0
```
BEGIN:VCARD
VERSION:4.0
UID:urn:uuid:4fbe8971-0bc3-424c-9c26-36c3e1eff6b1
FN:Simon Perreault
N:Perreault;Simon;;;ing. jr,M.Sc.
BDAY:--0203
ANNIVERSARY:20090808T1430-0500
GENDER:M
LANG;PREF=1:fr
LANG;PREF=2:en
ORG;TYPE=work:Viagenie
ADR;TYPE=work:;Suite D2-630;2875 Laurier;Quebec;QC;G1V 2M2;Canada
TEL;VALUE=uri;TYPE="work,voice";PREF=1:tel:+1-418-656-9254;ext=102
TEL;VALUE=uri;TYPE="work,cell,voice,video,text":tel:+1-418-262-6501
EMAIL;TYPE=work:simon.perreault@viagenie.ca
GEO;TYPE=work:geo:46.772673,-71.282945
KEY;TYPE=work;VALUE=uri:http://www.viagenie.ca/simon.perreault/simon.asc
TZ:-0500
URL;TYPE=home:http://nomis80.org
END:VCARD
```

### Group vCard
```
BEGIN:VCARD
VERSION:4.0
KIND:group
FN:The Doe family
MEMBER:urn:uuid:03a0e51f-d1aa-4385-8a53-e29025acd8af
MEMBER:urn:uuid:b8767877-b4a1-4c70-9acc-505d3819e519
END:VCARD
```

### Organization vCard
```
BEGIN:VCARD
VERSION:4.0
KIND:org
FN:ABC Marketing
ORG:ABC\, Inc.;North American Division;Marketing
END:VCARD
```

## MIME Type

- **Type**: `text/vcard`
- **File extensions**: `.vcf`, `.vcard`
- **Charset**: UTF-8 only (no other charset allowed)
- **Deprecated types**: `text/directory`, `text/x-vcard`
