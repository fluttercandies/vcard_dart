# RFC 2426: vCard MIME Directory Profile (vCard 3.0)

> **Document Type**: Standard Summary  
> **RFC Number**: 2426  
> **Category**: Standards Track  
> **Status**: Obsoleted by RFC 6350  
> **Date**: September 1998  
> **Authors**: F. Dawson (Lotus), T. Howes (Netscape)

## Overview

RFC 2426 defines the **vCard 3.0** format as a MIME Content-Type for directory information based on the [MIME-DIR] (RFC 2425) framework. It specifies how to represent electronic business card information.

## Key Requirements

### Required Properties

A vCard object **MUST** contain:
- `VERSION` - Value MUST be "3.0"
- `FN` - Formatted Name
- `N` - Structured Name

### Structure

```
BEGIN:VCARD
VERSION:3.0
... properties ...
END:VCARD
```

## Value Types

### Predefined Value Types (from RFC 2425)
- `uri` - URI reference
- `date` - Calendar date
- `date-time` - Date and time
- `float` - Floating point number
- `text` - Text string

### Extended Value Types (RFC 2426 additions)
- `binary` - Base64 encoded binary data (ENCODING=b)
- `vcard` - Embedded vCard
- `phone-number` - Telephone number (E.163/X.121)
- `utc-offset` - Timezone offset (e.g., +05:00, -08:00)

## Properties Reference

### Identification Properties

| Property | Cardinality | Value Type | Description |
|----------|-------------|------------|-------------|
| `FN` | 1 (REQUIRED) | text | Formatted name |
| `N` | 1 (REQUIRED) | structured | Family;Given;Additional;Prefix;Suffix |
| `NICKNAME` | * | text-list | Nickname(s) |
| `PHOTO` | * | binary/uri | Photograph |
| `BDAY` | * | date/date-time | Birthday |

### Delivery Addressing Properties

| Property | Cardinality | Value Type | Description |
|----------|-------------|------------|-------------|
| `ADR` | * | structured | PO Box;Ext;Street;City;Region;Postal;Country |
| `LABEL` | * | text | Formatted delivery address |

**ADR TYPE Parameters**: `dom`, `intl`, `postal`, `parcel`, `home`, `work`, `pref`  
**Default**: `TYPE=intl,postal,parcel,work`

### Telecommunications Properties

| Property | Cardinality | Value Type | TYPE Values |
|----------|-------------|------------|-------------|
| `TEL` | * | phone-number | `home`, `work`, `voice`, `fax`, `cell`, `video`, `pager`, `bbs`, `modem`, `car`, `isdn`, `pcs`, `pref`, `msg` |
| `EMAIL` | * | text | `internet`, `x400`, `pref` |
| `MAILER` | * | text | Email client identifier |

**Default TEL TYPE**: `voice`  
**Default EMAIL TYPE**: `internet`

### Geographical Properties

| Property | Cardinality | Value Type | Format |
|----------|-------------|------------|--------|
| `TZ` | * | utc-offset/text | `+/-HH:MM` or text |
| `GEO` | * | structured | `latitude;longitude` (6 decimal places) |

### Organizational Properties

| Property | Cardinality | Value Type | Description |
|----------|-------------|------------|-------------|
| `TITLE` | * | text | Job title |
| `ROLE` | * | text | Business category/function |
| `LOGO` | * | binary/uri | Organization logo |
| `AGENT` | * | vcard/uri/text | Representative |
| `ORG` | * | structured | Organization;Unit1;Unit2;... |

### Explanatory Properties

| Property | Cardinality | Value Type | Description |
|----------|-------------|------------|-------------|
| `CATEGORIES` | * | text-list | Categories |
| `NOTE` | * | text | Comments |
| `PRODID` | * | text | Product identifier |
| `REV` | * | date-time/date | Last revision |
| `SORT-STRING` | * | text | Sort key |
| `SOUND` | * | binary/uri | Audio clip |
| `UID` | * | text | Unique identifier |
| `URL` | * | uri | Website |
| `VERSION` | 1 (REQUIRED) | text | Must be "3.0" |

### Security Properties

| Property | Cardinality | Value Type | Description |
|----------|-------------|------------|-------------|
| `CLASS` | * | text | `PUBLIC`, `PRIVATE`, `CONFIDENTIAL` |
| `KEY` | * | binary/text | Public key/certificate |

**KEY TYPE**: `X509`, `PGP`, or IANA-registered type

## Parameters

### Common Parameters
- `LANGUAGE` - Language tag (RFC 1766)
- `ENCODING` - `b` for base64
- `VALUE` - Value type override
- `TYPE` - Property type qualifier

### Parameter Syntax
```
PROPERTY;PARAM1=value1;PARAM2=value2:value
PROPERTY;TYPE=work,voice,pref:value
```

## Escaping Rules

### Characters Requiring Escape

| Character | Escape Sequence |
|-----------|-----------------|
| `\` (backslash) | `\\` |
| `;` (semicolon) | `\;` |
| `,` (comma) | `\,` |
| Newline | `\n` or `\N` |

### Structured Value Delimiters
- **Component separator**: `;` (semicolon)
- **List separator**: `,` (comma)

## Line Folding

Lines longer than 75 characters SHOULD be folded:
- Insert CRLF followed by a single whitespace character (space or tab)
- Unfold by removing CRLF + following whitespace

## Binary Data Encoding

For PHOTO, LOGO, SOUND, KEY with inline data:
```
PHOTO;ENCODING=b;TYPE=JPEG:base64data...
```

- MUST use `ENCODING=b` parameter
- SHOULD fold long lines at 75 characters
- No blank line required after encoded data

## Differences from vCard 2.1

| Feature | vCard 2.1 | vCard 3.0 |
|---------|-----------|-----------|
| VERSION | 2.1 | 3.0 |
| Encoding | QUOTED-PRINTABLE, BASE64 | Only `b` (base64) |
| Newlines in text | Literal CRLF or QP | `\n` or `\N` escape |
| Escape characters | Not required | `;` and `,` MUST be escaped |
| TYPE prefix | Optional | Required (`TYPE=`) |
| CHARSET param | Allowed | Not allowed (use MIME header) |
| Required types | VERSION only | VERSION, FN, N |
| New types | - | CATEGORIES, CLASS, NICKNAME, PRODID, SORT-STRING |

## ABNF Grammar (Key Excerpts)

```abnf
vcard = [group "."] "BEGIN" ":" "VCARD" 1*CRLF
        1*(contentline)
        [group "."] "END" ":" "VCARD" 1*CRLF

contentline = [group "."] name *(";" param) ":" value CRLF

n-value = 0*4(text-value *("," text-value) ";")
          text-value *("," text-value)
          ; Family; Given; Middle; Prefix; Suffix

adr-value = 0*6(text-value ";") text-value
            ; PO Box; Ext; Street; City; Region; Postal; Country

ESCAPED-CHAR = "\\" / "\;" / "\," / "\n" / "\N"
```

## Examples

### Basic vCard 3.0
```
BEGIN:VCARD
VERSION:3.0
FN:John Q. Public
N:Public;John;Quinlan;Mr.;Esq.
END:VCARD
```

### Complete vCard 3.0
```
BEGIN:VCARD
VERSION:3.0
FN:Mr. John Q. Public\, Esq.
N:Public;John;Quinlan;Mr.;Esq.
NICKNAME:Jim,Jimmie
BDAY:1996-04-15
ADR;TYPE=dom,home,postal,parcel:;;123 Main Street;Any Town;CA;91921-1234
TEL;TYPE=work,voice,pref,msg:+1-213-555-1234
EMAIL;TYPE=internet,pref:jqpublic@xyz.dom1.com
ORG:ABC\, Inc.;North American Division;Marketing
TITLE:Director\, Research and Development
ROLE:Programmer
GEO:37.386013;-122.082932
TZ:-05:00
URL:http://www.example.com
NOTE:This is a note with special chars\; and commas\, escaped.
CATEGORIES:INTERNET,IETF,INDUSTRY
REV:1995-10-31T22:27:10Z
END:VCARD
```

## Implementation Notes

1. **Case Insensitivity**: Property names and parameter names are case-insensitive
2. **Grouping**: Properties can be grouped using a prefix (e.g., `HOME.TEL:...`)
3. **X- Extensions**: Custom properties prefixed with `X-` are allowed
4. **Multiple Instances**: Most properties can appear multiple times
5. **Empty Components**: Missing components in structured values must still include separators
